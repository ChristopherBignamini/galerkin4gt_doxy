Stiffness Matrix Discretization

The stiffness discretization is similar to the one for the Mass Matrix Discretization. We have to compute here the inverse of the transformation matrix besides its determinant (the jacobian).

Intuitively, let's call and the coordinates in the reference and actual frame of reference respectively: the derivative in the actual frame of reference reads where is the inverse of the transformation matrix ( ).

We will describe the example stiffness.cpp . As for the case of mass.cpp we define the discretization, consisting of polynomial order for the geometric map and for the approximation of the solution, and the cubature order. examples/stiffness.cpp definitions

We instantiate the quantities, and compute the values and derivatives of the basis funcitons on the quadrature points examples/stiffness.cpp instantiation Note: in mass.cpp we did not compute the derivatives of the basis functions used for the approximation, only the derivatives of the geometric map, which are needed in order to compute the Jacobian. Now we need both the derivatives, and we can skip the computation of the values of the basis funcitons.

We proceed then by instantiating the assembler examples/stiffness.cpp as_instantiation assembly_base contains the mesh, while assembly contains the storages for the other quantities (the transformation matrix, its determinant, its inverse). We define the stiffness matrix examples/stiffness.cpp instantiation_stiffness As for the case of the Mass Matrix Discretization, the block diagonal matrix is represented as a 5D array with the first three dimensions being the space dimensions, and the 2 extra dimensions being the number of rows and columns. It follows the definition of the "placeholders" to the storages, which are the arguments used in the definition of the computation. examples/stiffness.cpp placeholders Each placeholder is assigned to a storage with the definition of an "aggregator" examples/stiffness.cpp aggregator The aggregator defines the binding between the placeholders and the storages instantiated by the user, see the GridTools documentation for a detailed description of the aggregator concept. Eventually the computation tree is defined, instantiated and executed examples/stiffness.cpp computation With respect to the Mass Matrix Discretization example we notice that we have to compute the inverse of the Jacobian matrix.
